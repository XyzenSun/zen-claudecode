# zen-learn --status

## 任务目标

向用户展示当前学习状态

通过 subagent 完成所有读取、提炼、组织工作，主会话仅做前置检查与结果中继
**只读，不更改文件**

## 前置检查

1. 用 Bash 检查 `./zen-learn/status/progress.md` 是否存在（如 `test -f ./zen-learn/status/progress.md`）
2. 若不存在，不再进入工作流程，回复用户：您可能暂未开始学习，请先开始学习。

## 工作流程
### Step 1: 调起 subagent

用 `Agent` 工具发起一次调用（`subagent_type: Explore`），prompt 使用以下完整内容：

````
你是 zen-learn 学习状态汇报员。请完成以下任务，直接输出**给用户查看的 Markdown 文本**，不要夹带任何解释、前言或元评论。

## 读取以下文件

1. `./zen-learn/status/progress.md`
2. `./zen-learn/status/plan.md`
3. `./zen-learn/status/about_user.md`

## 信息提取

- 从 `plan.md` frontmatter 取：`topic` / `goal` / `expected_duration`
- 从 `progress.md` 取：
  - `current_stage`（frontmatter）
  - **已完成章节 / 总章节**：以"统计信息"段为准，不要重新核对
  - **小节分布**：扫描"进度总览"表的状态列计数（完成 / 进行中 / 未开始）
  - **总学习天数**：以"统计信息"段为准
  - **最近一次学习日期**：学习日志表最末一行
  - **最近 3 次学习**：学习日志表最末 3 行
  - 从 `about_user.md` 取：`learning_style` / `habits`（**仅用于鼓舞语的语气调整**，不展示）
  - 未读取到的信息，用「未知」代替

## 计算「距今」

执行 Bash `date +%Y-%m-%d` 取今天，与"最近一次学习日期"相减得到 X：

- X=0 显示"今天"
- X=1 显示"昨天"
- 其他显示"X 天前"

## 输出格式

```markdown

## zen-learn 学习状态

**主题**：{{topic}}
**目标**：{{goal}}
**预计时长**：{{expected_duration}}

### 总体进度
- 当前章节：{{current_stage}}
- 已完成章节：{{M_done}} / {{M_total}}
- 小节分布：完成 {{N_done}} · 进行中 {{N_ip}} · 未开始 {{N_todo}}
- 总学习天数：{{total_days}} 天
- 最近一次学习：{{last_date}}（{{距今表述}}）

### 最近学习记录

| 日期 | 学习内容 | 关键收获 |
|---|---|---|
| ... | ... | ... |

---

{{鼓舞语 1–3 句}}
```

若学习日志行数 < 3，有多少条展示多少条。不要输出"需关注项"段落。

## 鼓舞语生成要求

-引用具体进度数字、下一步小节名
- 参考 `about_user.md` ，了解用户的信息，调整语气与建议方向
- **按场景差异化**：

| 场景 | 策略 |
|---|---|
| 距今 ≥ 7 天未学 | 温和召回，建议先 `/zen-learn --continue` 回顾上次内容 |
| 已完成 > 80% | 冲刺鼓励，点名剩余关键小节 |
| 刚完成一个章节 | 祝贺并引出下一章主题 |
| 明显落后 `expected_duration` | 客观陈述差距，提议调整节奏或精简 step |
| 正常推进 | 简短点名下一步即可 |

**示范**：

- 「加油！继续努力！相信你一定可以！」
-  已完成 6 / 16 章，下一步是第七章 7.1 框架设计。按每天 1 小时的习惯大约还需 10 天完成，节奏合理。」

## 输出约束

- 直接输出上面格式的 Markdown 文本
- 不要加"好的"、"以下是"之类的前言
- 不要加"希望对你有帮助"之类的结尾
- 不要解释你做了什么
````

### Step 2: 返回结果

将subagent的回复不做任何修改，直接返回
